# MVP_OFFLINE input contract — `trades.csv` / `trades.parquet`

Этот контракт описывает **минимальный входной файл** для офлайн‑MVP пайплайна:

`Trade event → Signal → Sim entry/exit → PnL/metrics`

Файл может быть **CSV** или **Parquet**.

- CSV: разделитель `,`, **обязателен header** (имена колонок ниже).
- Parquet: те же имена колонок; `ts` желательно хранить как timestamp (лучше timezone‑aware UTC).

## 1) Обязательные колонки (минимум)

| column        | type (logical)   | example                          | notes |
|--------------|------------------|----------------------------------|------|
| `ts`         | datetime (UTC)   | `2025-01-01T00:00:10Z`           | ISO‑8601 UTC для CSV. |
| `wallet`     | string           | `Wallet_01`                      | Адрес или псевдо‑id кошелька. |
| `token_mint` | string           | `So11111111111111111111111111111111111111112` | Solana mint. |
| `side`       | enum string      | `buy` / `sell`                   | Строго нижний регистр. |
| `price`      | float (USD)      | `0.00091234`                     | Цена токена в USD на момент события. `> 0`. |
| `size_usd`   | float (USD)      | `125.50`                         | Оценка размера сделки в USD. `>= 0`. |

## 2) Необязательные колонки (можно добавлять без ломания контракта)

| column         | type   | example            | notes |
|----------------|--------|--------------------|------|
| `platform`     | string | `raydium`/`jupiter`/`pumpfun`/`meteora`/`other` | Сейчас используется только для разбивок. |
| `size_token`   | float  | `123456.789`        | Кол-во токенов (если известно). Можно держать `0`/пусто. |
| `tx_hash`      | string | `5uZ...`            | Идентификатор транзакции (если есть). |
| `slot`         | int    | `260123456`         | Solana slot (если есть). |
| `pool_address` | string | `9wFF...`           | Адрес пула/AMM (если доступно). |

## 3) Инварианты и ожидания пайплайна

- Пайплайн **генерирует сигналы только на `side == "buy"`**.
- Симулятор использует **будущие строки по тому же `token_mint`** как «тики цены».
  Поэтому для осмысленной симуляции в датасете должны быть **последующие события по тому же токену**
  (это может быть любая активность, не обязательно того же кошелька).
- Сортировка по `ts` должна быть стабильной (пайплайн сортирует, но лучше подавать уже отсортированным).

## 4) Пример датасета‑заглушки

- CSV: `data/processed/trades_stub.csv` (входит в pack)
- Parquet: можно сгенерировать локально из CSV:

```bash
python tools/convert_csv_to_parquet.py \
  --csv data/processed/trades_stub.csv \
  --parquet data/processed/trades_stub.parquet
```

> Для `to_parquet()` нужен backend (`pyarrow`).


## 3) Опциональные поля оверлеев (не обязательны для MVP)

Если вы уже рассчитали Polymarket/Regime‑overlay (внешний сентимент), можно хранить его отдельно
и джойнить по времени к сигналам. Для простоты допускается колонка в `trades.*`:

| column        | type (logical) | example | notes |
|--------------|-----------------|---------|------|
| `risk_regime` | float           | `0.35`  | Скаляp режима `[-1..+1]` (risk‑on>0 / risk‑off<0). Если нет — считается `0`. |
